## Design System

| Property | Value |
|----------|-------|
| Tool | Streamlit (.streamlit/config.toml `[theme]`) |
| Preset | Neo4j-inspired palette (user decision) |

## Spacing Scale

Streamlit controls spacing via `st.columns`, `st.container`, and markdown spacing. No CSS tokens needed. The following conventions apply:

| Pattern | Usage | Streamlit Mechanism |
|---------|-------|---------------------|
| Inline gap | Between icon and label | Single space in markdown |
| Element gap | Between form fields | Streamlit default (automatic) |
| Section break | Between logical sections | `st.divider()` or `st.markdown("---")` |
| Tab content padding | Inside each tab | `st.container()` (Streamlit default) |
| Column gap | Between side-by-side elements | `st.columns(gap="medium")` |

Exceptions: None. Streamlit handles all spacing internally.

---

## Typography

Streamlit controls typography via the theme font setting. Custom sizes are set via markdown headers.

| Role | Streamlit Element | Usage |
|------|-------------------|-------|
| Page title | `st.title()` | One per page, top of page |
| Section heading | `st.subheader()` | Major sections within a page |
| Label | `st.caption()` | Metadata labels, timestamps, secondary info |
| Body | Default `st.write()` / `st.markdown()` | All other text |

Rules:
- Never use `st.header()` -- use `st.subheader()` for section headings to avoid visual bloat.
- Use `st.caption()` for all timestamps, IDs, and secondary metadata.
- Bold text via `**text**` for emphasis within body copy only.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F5F5F0` | Page background |
| Secondary (30%) | `#E8E8E0` | Sidebar, expander backgrounds, card-like containers |
| Accent (10%) | `#68BF40` | Primary buttons, active tab indicator, selected sidebar item, success badges |
| Destructive | `#D32F2F` | Soft-delete buttons, warning banners for dependency conflicts |
| Warning | `#F9A825` | Integrity check warnings, stale experiment indicators |

Accent reserved for:
- Primary action buttons ("Save Model", "Upload Recipe", "Apply Rollback", "Confirm Squash")
- Active navigation indicator (sidebar)
- Success status badges (COMPLETED)
- Graph node highlight on selection (streamlit-agraph)

Color via Streamlit (no custom CSS):
- `st.success()` for success messages (uses green)
- `st.error()` for destructive/error messages (uses red)
- `st.warning()` for warnings (uses amber)
- `st.info()` for informational messages (uses blue)

## Page Structure

### Navigation (Sidebar)

```
Sidebar
  Models
  Recipes
  Components
  Experiments
  Checkpoints        [NEW]
  ---
  Graph View         [NEW]
  History            [NEW]
  ---
  Admin Console      [NEW]
```

Separator via `st.sidebar.markdown("---")` between entity pages, visualization pages, and admin.

### Page Layout Pattern

Every entity page follows this structure:

```
st.title("Entity Name")
tab_browse, tab_create = st.tabs(["Browse", "Create"])  # or ["Browse", "Upload"] for Recipe

with tab_browse:
    # Filters row: st.columns([2, 2, 1])
    #   col1: search/filter
    #   col2: status filter (selectbox)
    #   col3: action button (if applicable)
    # Results: st.dataframe() or st.expander() list

with tab_create:
    # st.form() with submit button
```

For read-only entities (Experiment, Checkpoint): tab order is `["Browse"]` only, with inline edit via `st.expander()`.

### Error States

| Context | Message |
|---------|---------|
| DB connection failed | "Cannot connect to Neo4j. Check that the database is running and connection settings are correct." |
| Entity not found | "{Entity} not found. It may have been deleted or the ID is incorrect." |
| Duplicate name/URI | "A {entity} with this name already exists. Use the upsert option to update it." |
| Integrity violation | "Cannot delete: {N} experiments depend on this {entity}." |
| URI unreachable | "The URI path could not be verified. Confirm the path exists and is accessible." |

### Destructive Action Confirmations

| Action | Confirmation Pattern |
|--------|---------------------|
| Soft-delete Experiment | `st.warning("This will hide the experiment from browse views.")` + `st.checkbox("I understand this experiment will be hidden")` + `st.button("Hide Experiment")` |
| Soft-delete Checkpoint | `st.warning("N experiments use this checkpoint as a starting point.")` (if deps exist) + `st.checkbox("I understand dependent experiments may be affected")` + `st.button("Hide Checkpoint")` |
| Checkpoint URI edit | Wizard: show current URI -> input new -> `st.info("This will update the URI for checkpoint {name}")` + `st.checkbox("I confirm this URI change")` + `st.button("Update URI")` |
| Rollback | `st.warning("This will revert the experiment chain to the selected point.")` + preview diff + `st.checkbox("I have reviewed the changes above")` + `st.button("Apply Rollback")` |
| Squash | `st.warning("This will merge {N} experiments into a single node. This cannot be undone.")` + preview + `st.checkbox("I have reviewed the squash preview")` + `st.button("Confirm Squash")` |

## Graph Visualization Contract (US-7)

### streamlit-agraph Configuration

```python
from streamlit_agraph import agraph, Node, Edge, Config

config = Config(
    width=800,
    height=600,
    directed=True,
    physics=True,
    hierarchical=Config.Hierarchical(
        enabled=True,
        direction="UD",      # top-down DAG
        sortMethod="directed"
    )
)
```

## Wizard Flows

### Checkpoint URI Edit Wizard (US-6)

```
Step 1: st.selectbox("Select checkpoint")
        st.text_input("Current URI", disabled=True, value=current_uri)
Step 2: st.text_input("New URI")
Step 3: st.info("Dependencies: {N} experiments started from this checkpoint")
        st.dataframe(dependent_experiments)  # if any
Step 4: st.checkbox("I confirm this URI change")
        st.button("Update URI")
```

### Rollback Wizard (US-9)

```
Step 1: st.selectbox("Select experiment to rollback to")
Step 2: st.subheader("Preview")
        st.json(diff_preview)  # show what will change
        st.caption("Experiments after this point will be hidden (usable=false)")
Step 3: st.checkbox("I have reviewed the changes above")
        st.button("Apply Rollback")
```

### Squash Wizard (US-10)

```
Step 1: st.selectbox("From experiment")
        st.selectbox("To experiment")
Step 2: st.subheader("Squash Preview")
        st.write(f"Merging {N} experiments into one")
        st.dataframe(experiments_to_squash)
Step 3: st.checkbox("I have reviewed the squash preview")
        st.button("Confirm Squash")
```
