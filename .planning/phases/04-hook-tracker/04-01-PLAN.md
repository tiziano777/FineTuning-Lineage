## Wave 2: Tracker decorator + PRE/POST lifecycle

### Task 2.1: Create `graph_lineage/lineage/tracker.py`
```python
class TrackerConfig:
    blocking: bool = True
    config_path: str  # extracted from decorated function args

class envelope:
    @staticmethod
    def tracker(blocking: bool = True):
        def decorator(fn):
            @functools.wraps(fn)
            def wrapper(*args, **kwargs):
                ctx = _pre_execution(args, kwargs, blocking)
                if ctx is None:
                    # non-blocking mode, PRE failed
                    return fn(*args, **kwargs)
                try:
                    result = fn(*args, **kwargs)
                    _post_execution(ctx, status="COMPLETED")
                    return result
                except Exception as e:
                    _post_execution(ctx, status="FAILED", exit_msg=str(e))
                    raise
            return wrapper
        return decorator
```

### Task 2.2: Implement `_pre_execution()`
Steps:
1. Extract `config_path` from first positional arg
2. Load YAML → `LineageConfig`
3. Resolve `StorageProvider` (local)
4. `validate_pre_execution()` — if errors and blocking → `sys.exit(2)`
5. `CodebaseSnapshot.capture_snapshot(project_root)`
6. Query Neo4j for parent experiment (by URI)
7. `detect_run_type(config, snapshot, parent)`
8. Create Experiment node in Neo4j (strategy, hashes, codebase)
9. Create edges (DERIVED_FROM/RETRY_OF/STARTED_FROM based on strategy)
10. `ConfigWriter.write_back()` — inject generated experiment.id into config.yml
11. Return `ExecutionContext(exp_id, strategy, ...)`

If any step fails:
- `blocking=True` → log error, exit with appropriate code
- `blocking=False` → log warning, return None

### Task 2.3: Implement `_post_execution()`
Steps:
1. Update Experiment node: status + exit_status + exit_msg
2. If COMPLETED: read metrics URIs from config, store in node
