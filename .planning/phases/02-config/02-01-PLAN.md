# PLAN 02-01 — Config Schema & Validation

**Phase**: 2 — Config Schema & Validation
**Goal**: Pydantic v2 models for config.yml parsing, validation, and atomic write-back
**Depends on**: Phase 1 (StorageProvider)

---

## Decisions (from discussion)

1. **Comments only** to separate managed vs editable zones (no hash/integrity mechanism)
2. **MERGE is standalone**: separate project with `model_merging.enabled=true`, always uses tracker
3. **Schema ibrido**: `experiment` and `recipe` are strict Pydantic models; `model`, `hardware` are `dict[str, Any]` for framework flexibility
4. **Pre-train write**: `experiment.id` written immediately + `previous_experiment_id` + `base_experiment_id` + `base: bool`
5. **Recipe guard**: recipe snapshot stored at experiment level; if recipe content changes pre-train → exit_code=5 (logic conflict), system blocks

---

## Config.yml Structure

```yaml
# ============================================================
# LINEAGE-MANAGED — Do not edit fields below manually
# ============================================================
experiment:
  id: null                            # UUID auto-generated pre-train
  previous_experiment_id: null        # UUID of immediate parent experiment
  base_experiment_id: null            # UUID of t_0 experiment (root of lineage)
  base: true                          # true = this is t_0, false = evolved from base
  name: "my_experiment"               # REQUIRED, human-readable
  description: ""                     # optional
  uri: "/path/to/project"             # REQUIRED, project root
  status: null                        # RUNNING | COMPLETED | FAILED (managed by hook)
  checkpoint_resume_from: null        # ckp_id for RESUME/BRANCH trigger

# ============================================================
# USER-DEFINED — Edit freely
# ============================================================
model:                                # dict[str, Any] — framework-dependent
  model_name: "llama-7b"              # REQUIRED (min key for tracking)
  framework: "pytorch"                # REQUIRED (min key for tracking)
  # ... any other keys (hyperparams, quantization, etc.)

recipe:                               # strict — snapshot for consistency guard
  id: UUID
  name: str
  description: str
  scope: str
  tags: list
  derived_from: UUID
  entries: -                        # snapshot copy from Recipe entity

output:
  output_dir: "./outputs"             # REQUIRED
  metrics_uri: null                   # auto-filled post-train
  hw_metrics_uri: null                # auto-filled post-train

hardware: -                         # dict[str, Any] — auto-detected or user-specified

# ============================================================
# MERGE-ONLY (required only if this project is a merge experiment)
# ============================================================
model_merging:
  enabled: false
  merge_method: null                  # e.g. "linear", "slerp", "ties"
  sources: -                         # list of ckp_id or model_name
```

## Tasks

### Task 1: Create Pydantic config models

**Files**: `graph_lineage/config/data_classes/*.py`

Models:

```python

# experiment_config.py
class ExperimentConfig(BaseModel):
    """Strict — all fields defined, managed by hook."""
    id: str | None = None
    previous_experiment_id: str | None = None
    base_experiment_id: str | None = None
    base: bool = True
    name: str = Field(..., min_length=1)
    description: str = ""
    uri: str = Field(..., min_length=1)
    status: str | None = None
    checkpoint_resume_from: str | None = None

# recipe_config.py
class RecipeEntry(BaseModel):
    """Metadata for a single distribution/dataset entry in a recipe."""

    chat_type: str = Field(..., min_length=1, description="Chat conversation type")
    dist_id: str = Field(..., min_length=1, description="Distribution unique identifier")
    dist_name: str = Field(..., min_length=1, description="Human-readable distribution name")
    dist_uri: str = Field(..., min_length=1, description="Path or URI to distribution")
    replica: int = Field(1, ge=1, description="Replication factor (N× oversampling)")
    samples: int = Field(..., gt=0, description="Total number of samples in distribution")
    system_prompt: list[str] | None = Field(None, description="System prompt templates")
    system_prompt_name: list[str] | None = Field(None, description="System prompt names")
    tokens: int = Field(..., gt=0, description="Total token count")
    words: int = Field(..., gt=0, description="Total word count")
    validation_error: str | None = Field(None, description="Validation error if any")


class RecipeConfig(BaseModel):
    """Full recipe configuration — preserves all metadata fields from the YAML.

    Only 'entries' is required; all other metadata fields are optional and non-blocking.
    The recipe will process successfully even if metadata is missing.
    """

    entries: dict[str, RecipeEntry] = Field(
        ...,
        description="Mapping of dataset paths to distribution metadata (REQUIRED)"
    )
    recipe_id: str | None = Field(None, description="Recipe UUID")
    recipe_name: str | None = Field(None, description="Recipe short name")
    description: str | None = Field(None, description="Human-readable description")
    scope: str | None = Field(None, description="Training scope (e.g. continual_ft, sft)")
    tasks: list[str] = Field(default_factory=list, description="Task categories covered")
    tags: list[str] = Field(default_factory=list, description="Free-form classification tags")
    derived_from: str | None = Field(None, description="Parent recipe UUID this was derived from")

# model_merging.py
class ModelMergingConfig(BaseModel):
    """Optional — only required when enabled=true."""
    enabled: bool = False
    merge_method: str | None = None
    sources: list[str] = Field(default_factory=list)

# output_config.py
class OutputConfig(BaseModel):
    output_dir: str = Field(..., min_length=1)
    metrics_uri: str | None = None
    hw_metrics_uri: str | None = None

# lineage_config.py
class LineageConfig(BaseModel):
    """Root config — strict on experiment/recipe, flexible on model/hardware."""
    experiment: ExperimentConfig
    model: dict[str, Any] = Field(default_factory=dict)
    recipe: RecipeConfig = Field(default_factory=RecipeConfig)
    output: OutputConfig
    hardware: dict[str, Any] = Field(default_factory=dict)
    model_merging: ModelMergingConfig = Field(default_factory=ModelMergingConfig)
```

Validation rules at model level:
- `model` dict MUST contain `model_name` (str, non-empty) and `framework` (str, non-empty)
- Use `@model_validator` on `LineageConfig` to enforce minimum model keys

### Task 3: Create atomic config writer

**File**: `graph_lineage/config/writer.py`

```python
def load_config(path: str, storage: StorageProvider) -> LineageConfig
def save_config(config: LineageConfig, path: str, storage: StorageProvider) -> None
```

Atomic write:
1. `storage.backup(path)` → creates `.bak.TIMESTAMP` copy
2. Serialize via `yaml.dump` (pyyaml, preserving dict ordering)
3. Write to `path + ".tmp"`
4. `os.rename(tmp, path)` (atomic on POSIX)

Comment injection: prepend managed/user zone comments to output YAML string.

**Verify**: Round-trip load→modify→save→reload; backup file exists.
